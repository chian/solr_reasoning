INSTRUCTIONS:
- Use <think> tags to explain your reasoning process
- Use <action> tags to make MCP tool calls
- Each <action> tag should contain a JSON object with:
    - "tool_name": the name of the MCP tool to call (see list above)
    - "parameters": a dictionary of arguments for that tool
- If multiple tool calls are needed, put each in its own <action> tag
- For advanced operations (like feature extraction), use run_p3_tool with the appropriate p3-tool command string
- Only use tool names that are listed above
- After each action is executed, an <observation> tag will be inserted with the results
- You can update your reasoning and future actions based on these observations

FORMAT EXAMPLE:
<think>
First, I need to understand what the user is asking for. They want to find genomes associated with Escherichia coli. I should use the find_similar_genomes tool to search for genomes with genus "Escherichia" and species "coli".
</think>

<action>
{
  "tool_name": "find_similar_genomes",
  "parameters": {
    "genus": "Escherichia",
    "species": "coli",
    "limit": 20
  }
}
</action>

<observation>
Action result: {"status": "success", "genomes": [{"genome_id": "12345.1", "genus": "Escherichia", "species": "coli"}]}
</observation>

<think>
Now that I have the genome IDs, I can extract features from the first genome to get more detailed information.
</think>

<action>
{
  "tool_name": "run_p3_tool",
  "parameters": {
    "command": "p3-get-genome-features --eq genome_id,12345.1 --attr feature_id,product"
  }
}
</action>
