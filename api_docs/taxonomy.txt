Taxonomy record: Represents a taxonomic node and associated metadata in BV-BRC.

A taxonomy record describes a taxonomic node, including identifiers, lineage, genome statistics, and metadata. Taxonomy data are used for classification, annotation, and comparative genomics.

Fields:
- taxon_id: NCBI Taxonomy ID. Example: 83332
- taxon_name: Name of the taxon. Example: "Escherichia coli"
- taxon_rank: Rank of the taxon. Example: "species"
- parent_id: Parent taxon ID. Example: 562
- lineage: Taxonomic lineage string. Example: "Bacteria; Proteobacteria; ..."
- lineage_ids: List of lineage taxon IDs. Example: [83332, 562]
- lineage_names: List of lineage names. Example: ["Bacteria", "Proteobacteria"]
- lineage_ranks: List of lineage ranks. Example: ["superkingdom", "phylum"]
- other_names: List of other names for the taxon. Example: ["coli"]
- division: Division of the taxon. Example: "Bacteria"
- genetic_code: Genetic code. Example: 11
- description: Description of the taxon. Example: "Gram-negative bacterium"
- core_families: Number of core families. Example: 2000
- core_family_ids: List of core family IDs. Example: ["PLF_000001"]
- genome_count: Number of genomes. Example: 100
- genome_length_mean: Mean genome length. Example: 4641652
- genome_length_sd: Standard deviation of genome length. Example: 10000
- genomes: Number of genomes. Example: 100
- genomes_f: Genomes field (meaning unclear). Example: "100F"
- cds_mean: Mean number of CDS. Example: 4300
- cds_sd: Standard deviation of CDS. Example: 50
- hypothetical_cds_ratio_mean: Mean ratio of hypothetical CDS. Example: 0.02
- hypothetical_cds_ratio_sd: Standard deviation of hypothetical CDS ratio. Example: 0.01
- plfam_cds_ratio_mean: Mean PLfam CDS ratio. Example: 0.85
- plfam_cds_ratio_sd: Standard deviation of PLfam CDS ratio. Example: 0.02
- _version_: Version number of the record. Example: 1
- taxon_id_i: Integer taxon ID. Example: 83332

Notes:
- Taxonomy records are used for classification, annotation, and comparative genomics.
- Not all fields are present for every record; see BV-BRC documentation for more details.
