Subsystem record: Represents a biological subsystem and associated metadata in BV-BRC.

A subsystem record describes a biological subsystem, including identifiers, roles, features, and metadata. Subsystems are used for functional annotation and comparative genomics.

Fields:
- id: Unique identifier for the subsystem. Example: SUB_00001
- subsystem_id: Subsystem identifier. Example: "SS_00001"
- subsystem_name: Name of the subsystem. Example: "Glycolysis"
- class: Class of the subsystem. Example: "Metabolism"
- subclass: Subclass of the subsystem. Example: "Carbohydrate metabolism"
- superclass: Superclass of the subsystem. Example: "Biological process"
- role_id: Role identifier. Example: "ROLE_00001"
- role_name: Name of the role. Example: "Hexokinase"
- feature_id: Feature identifier. Example: fig|83332.12.peg.1
- gene: Gene symbol. Example: dnaN
- genome_id: Identifier of the parent genome. Example: 83332.12
- genome_name: Name of the parent genome. Example: "Escherichia coli K12"
- product: Product name or description. Example: "DNA polymerase III subunit beta"
- patric_id: PATRIC-specific feature ID. Example: fig|83332.12.peg.1
- refseq_locus_tag: RefSeq locus tag. Example: "b0001"
- active: Whether the subsystem is active. Example: "yes"
- public: Whether the record is public. Example: true
- owner: Owner of the record. Example: "user@bv-brc.org"
- taxon_id: NCBI Taxonomy ID. Example: 83332
- date_inserted: Date the record was inserted. Example: 2020-01-01
- date_modified: Date the record was last modified. Example: 2021-01-01
- _version_: Version number of the record. Example: 1
- user_read: Users with read access. Example: ["user1", "user2"]
- user_write: Users with write access. Example: ["user1"]

Notes:
- Subsystems are used for functional annotation and comparative genomics.
- Not all fields are present for every record; see BV-BRC documentation for more details.
