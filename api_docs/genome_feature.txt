Genome feature record: Represents an annotated feature (gene, CDS, RNA, etc.) within a genome.

A genome feature is an individual biological element annotated on a genome, such as a gene, coding sequence (CDS), rRNA, tRNA, or other functional element. Each feature record contains identifiers, location, type, annotation source, and biological function.

Fields:
- feature_id: Unique identifier for the feature. Example: fig|83332.12.peg.1
- genome_id: Identifier of the parent genome. Example: 83332.12
- annotation: Source of the annotation (e.g., PATRIC, RefSeq, GenBank). Example: PATRIC
- feature_type: Type of feature (e.g., CDS, gene, rRNA, tRNA, misc_feature). Example: CDS
- start: Start coordinate of the feature. Example: 190
- end: End coordinate of the feature. Example: 2550
- strand: DNA strand (+ or -). Example: +
- product: Name or description of the feature's biological function. Example: DNA polymerase III subunit beta
- gene: Gene symbol or name (if available). Example: dnaN
- aliases: Alternative names or locus tags. Example: b0084, JW0073
- ec_number: Enzyme Commission number (if applicable). Example: 2.7.7.7
- figfam_id: FIGfam protein family ID (if applicable). Example: FIG00000002
- protein_id: Protein accession (if applicable). Example: NP_414543.1
- sequence: Nucleotide or protein sequence (if requested).
- aa_length: Length of the protein in amino acids. Example: 350
- aa_sequence_md5: MD5 checksum of the protein sequence. Example: 1bc29b36f623ba82aaf6724fd3b16718
- accession: Accession number for the feature. Example: NC_000913.3
- alt_locus_tag: Alternative locus tag. Example: b0001
- brc_id: BRC-specific identifier. Example: BRC12345
- classifier_round: Classifier round (for AMR features). Example: 1
- classifier_score: Classifier score (for AMR features). Example: 0.98
- codon_start: Start codon position. Example: 1
- date_inserted: Date the feature was inserted. Example: 2020-01-01
- date_modified: Date the feature was last modified. Example: 2021-01-01
- gene_id: Numeric gene identifier. Example: 123456
- genome_name: Name of the parent genome. Example: Escherichia coli K12
- go: Gene Ontology terms. Example: ["GO:0005737"]
- location: Genomic location string. Example: "NC_000913.3:190..2550"
- na_length: Length of the nucleotide sequence. Example: 1361
- na_sequence_md5: MD5 checksum of the nucleotide sequence. Example: 098f6bcd4621d373cade4e832627b4f6
- notes: Additional notes. Example: ["predicted by RAST"]
- og_id: Orthologous group ID. Example: OG_1234
- owner: Owner of the feature record. Example: user@bv-brc.org
- p2_feature_id: Secondary feature ID. Example: 78910
- patric_id: PATRIC-specific feature ID. Example: fig|83332.12.peg.1
- pdb_accession: PDB accession(s) for structure. Example: ["1A2B"]
- pgfam_id: Cross-genus protein family ID. Example: PGF_000001
- plfam_id: Genus-specific protein family ID. Example: PLF_000001
- property: Properties of the feature. Example: ["hypothetical"]
- public: Whether the feature is public. Example: true
- refseq_locus_tag: RefSeq locus tag. Example: "b0001"
- segments: Segments of the feature. Example: ["segment1", "segment2"]
- sequence_id: Sequence identifier. Example: "NC_000913.3"
- sog_id: Sub-orthologous group ID. Example: SOG_1234
- taxon_id: NCBI Taxonomy ID. Example: 83332
- uniprotkb_accession: UniProtKB accession. Example: P0A8V2
- user_read: Users with read access. Example: ["user1", "user2"]
- user_write: Users with write access. Example: ["user1"]

Notes:
- The 'annotation' field indicates the source of the annotation, not the biological function.
- The 'product' field contains the functional name or description of the feature (e.g., protein name).
- The 'gene' field is the gene symbol, if available.
- Not all fields are present for every feature; see BV-BRC documentation for more details.
