Structured assertion record: Represents a structured assertion and associated metadata in BV-BRC.

A structured assertion record describes an assertion about a biological feature, including evidence, property, value, and metadata. Structured assertions are used for annotation and data integration.

Fields:
- id: Unique identifier for the structured assertion. Example: SA_00001
- feature_id: Feature identifier. Example: fig|83332.12.peg.1
- patric_id: PATRIC-specific feature ID. Example: fig|83332.12.peg.1
- refseq_locus_tag: RefSeq locus tag. Example: "b0001"
- property: Property being asserted. Example: "antibiotic resistance"
- value: Value of the property. Example: "resistant"
- evidence_code: Evidence code for the assertion. Example: "IEA"
- comment: Additional comment. Example: "predicted by RAST"
- pmid: PubMed ID for supporting publication. Example: "12345678"
- score: Score for the assertion. Example: "0.95"
- source: Source of the assertion. Example: "RefSeq"
- owner: Owner of the record. Example: "user@bv-brc.org"
- public: Whether the record is public. Example: true
- user_read: Users with read access. Example: ["user1", "user2"]
- user_write: Users with write access. Example: ["user1"]
- date_inserted: Date the record was inserted. Example: 2020-01-01
- date_modified: Date the record was last modified. Example: 2021-01-01
- _version_: Version number of the record. Example: 1

Notes:
- Structured assertions are used for annotation and data integration.
- Not all fields are present for every record; see BV-BRC documentation for more details.
